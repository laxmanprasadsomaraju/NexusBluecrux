B. Full build prompt — paste this to build the tool
This is the complete prompt to use when building NEXUS. It is written to be given to a developer, a no-code tool, or an AI code generator. Every screen, every component, every interaction is specified.
NEXUS — Full Build Prompt
Build a web application called NEXUS — a Supply Chain Exception and Escalation Hub for Bluecrux.
TECH STACK: React + TypeScript frontend. Python + FastAPI backend. PostgreSQL database. Azure deployment. Microsoft Entra ID for SSO. No mock data — build with a proper data layer from the start.
DESIGN SYSTEM: Follow the exact colour tokens and typography rules in Section A of this document. The app must feel like a premium enterprise SaaS tool — clean, flat, no gradients, no drop shadows except on modals. Every colour must have a purpose. Never use red for anything except Critical exceptions.
LAYOUT: Fixed left sidebar (220px wide, background #0D1B2A) + main content area (remaining width, background #F8F9FA). Top bar inside each screen (white, 56px tall, 0.5px bottom border #CED4DA).

C2. Exception queue interactions
User action
What happens
Click stat card &apos;Critical open&apos;
Applies filter: severity=Critical. Filter pill &apos;Critical&apos; becomes active. List updates.
Click stat card &apos;Awaiting action&apos;
Applies filter: status=Awaiting action. No pill changes — filter is shown as a tag above the list: &apos;Filter: Awaiting action ✕&apos;.
Click filter pill &apos;All&apos;
Clears all filters. Shows full exception list. Pill &apos;All&apos; becomes active.
Click filter pill &apos;Critical&apos;
Filters list to severity=Critical only. Pill &apos;Critical&apos; becomes active.
Click filter pill &apos;Mine&apos;
Filters list to exceptions where owner = current user. Pill &apos;Mine&apos; becomes active.
Click filter pill &apos;Partner&apos;
Filters list to exceptions involving a CMO, supplier, or 3PL. Pill &apos;Partner&apos; becomes active.
Click exception card
Card becomes selected (border-left 2px #00B4D8, background changes). Detail panel loads that exception&apos;s data. Previous selected card deselects.
Click &apos;Raise exception&apos; button
Opens Raise exception modal (see B7).
Click &apos;Sync&apos; button
Icon spins. Sub-title shows &apos;Syncing…&apos;. API call to refresh data from all source systems. On complete: icon stops, sub-title shows &apos;Last synced just now&apos;.
C3. Detail panel interactions
User action
What happens
Click &apos;Approve [action]&apos; (primary button)
Shows confirmation tooltip: &apos;Confirm approval?&apos; with Yes/Cancel. On Yes: action logged in timeline with user name + timestamp. Exception status updates. API call pushes decision to source system. Button becomes disabled and shows &apos;Approved ✓&apos; text. Toast appears.
Click &apos;Escalate&apos;
Opens escalation form as a modal: &apos;Escalate to&apos; name search, deadline date-time, note text area. Submit sends notification to escalation target. Logs in timeline. Exception gets &apos;Escalated&apos; status badge.
Click &apos;Add note&apos;
Opens a small text area below the timeline (inline, not a modal). Submit adds note to timeline immediately. Note shows author name and timestamp.
Click &apos;Open in Axon&apos; / &apos;Open in Binocs&apos; etc.
Opens source system URL in a new browser tab. Deep link to the specific record.
Hover AI suggestion box
Tooltip appears: &apos;This suggestion is generated by AI. Always verify before approving.&apos; This reinforces that it is a suggestion, not an automatic action.
C4. My actions interactions
User action
What happens
Click &apos;Approve&apos; on action card
Same confirmation + timeline logging flow as detail panel approve. Card moves to a &apos;Completed today&apos; section at the bottom.
Click &apos;Review&apos; on action card
Opens the full exception detail as a side panel overlay (slides in from right, 480px wide). Same detail panel content.
Click &apos;Start&apos; on action card
Same as Review — opens the detail panel. Marks action status as &apos;In progress&apos;.
C5. Raise exception modal interactions
User action
What happens
Type in &apos;Owner&apos; field
Autocomplete dropdown appears after 2 characters. Shows matching users with their role subtitle. Arrow keys to navigate, Enter to select.
Select severity &apos;Critical&apos;
Severity dropdown shows red dot next to &apos;Critical&apos;. Selected value shown with severity dot in the field.
Click &apos;Submit exception&apos;
Validates all required fields (title, severity, owner). If missing: shows inline error in red below that field, shakes the submit button once. If valid: creates exception, closes modal, shows success toast, exception appears at top of queue with &apos;New&apos; indicator for 5 seconds.
Click &apos;Cancel&apos; or overlay
Closes modal. If any field has been filled, shows &apos;Discard changes?&apos; confirmation with Keep/Discard buttons.

Key rules — never break these
Never use red for anything except Critical severity exceptions. Red is reserved.
Never add drop shadows to cards or panels. Flat surfaces only. (Exception: modals get box-shadow: 0 20px 60px rgba(0,0,0,0.2))
Never use gradients anywhere in the UI.
Never use ALL CAPS on button labels. Sentence case only.
Never make badge text black on a coloured badge. Always use the same colour family as the badge background.
Never skip the confirmation step before a primary action (approve, escalate). Always ask &apos;Confirm?&apos; first.
Never let an AI suggestion appear as a confirmed action. The purple box is always clearly labelled &apos;AI suggestion&apos;.
Never show more than 4 action buttons in the action bar. If you need more, put them in a &apos;More actions&apos; dropdown.
The audit timeline is append-only. Nothing can be deleted from it. Ever.
Remember why these rules exist:
NEXUS is used by supply chain planners and directors at pharma companies like AstraZeneca, biotech companies like Argenx, and CPG companies like Maesa. These are regulated, high-stakes environments.
A visual mistake (showing green when something is critical, making an AI suggestion look like an approved action) can cause real operational harm.
Consistency is not an aesthetic preference — it is a safety feature.
